<objective>
Merge the old separate `WorkflowPR` (in-place branching) and `WorkflowWorktree` (isolated worktree) code paths in `processor.go` into a single unified implementation. After this prompt, `workflow: pr` always creates a git worktree, commits inside it, pushes, creates a PR, then removes the worktree — exactly what the old `WorkflowWorktree` did. No in-place PR branching code exists anymore.
</objective>
