<summary>

- Adds a precommit gate that scans the backend-neutral packages for container-flavored names (`containerName`, `ContainerChecker`, `ContainerStopper`, `containerslot`) and fails the build if any of them re-leak into the abstraction layer.
- Models the script on the existing `hotpath-logcheck.sh` / `hotpath-execcheck.sh` gates: a `warn`/`strict` mode arg, skips for test and generated files, and a clear offender list on stderr.
- The gate runs in strict mode from `make precommit` so a regression cannot merge.
- Ships a leak fixture that demonstrates the failure mode (a forbidden token in a neutral package) so the gate's detection is itself tested.
- Exits 0 on the migrated tree (after prompts 1 and 2 land); exits non-zero if a forbidden token reappears.

</summary>

## 1. Create `scripts/hotpath-execution-naming-check.sh`

Create `/workspace/scripts/hotpath-execution-naming-check.sh` modelled on `scripts/hotpath-execcheck.sh`, with these specifics:

1.1. Header comment: explain it scans the backend-neutral packages for container-flavored tokens that must not re-leak into the abstraction layer (spec 102). Document `warn` (print offenders, exit 0) and `strict` (print offenders, exit 1 if any) modes. Note that container vocabulary is allowed ONLY inside `pkg/executor/`, `pkg/launchpolicy/`, and `pkg/containerlock/` (the docker-CLI implementation).

1.2. `MODE="${1:-warn}"`, `ROOT=$(cd "$(dirname "$0")/.." && pwd)`, `cd "$ROOT"`.

1.3. Scan scope ŌĆö the neutral-layer packages ONLY. Iterate over exactly these directories' `*.go` files:
```
pkg/factory pkg/runner pkg/promptresumer pkg/cancellationwatcher pkg/queuescanner pkg/healthcheckgate pkg/generator pkg/processor pkg/executionslot
```
Use a loop like `for dir in <list>; do find "$dir" -type f -name '*.go' ...; done`, or `find pkg/factory pkg/runner ... -type f -name '*.go'`. Do NOT scan `pkg/executor`, `pkg/launchpolicy`, or `pkg/containerlock` ŌĆö those are the docker-CLI implementation and legitimately use container vocabulary.

1.4. Per-file skips (apply before grepping, `continue` on match):
   - Skip `*_test.go` files. (Rationale: the AC greps specify "non-test matches"; test files may legitimately reference renamed mock type names during transition.)
   - Skip counterfeiter-generated files (`head -3 "$f" | grep -q "Code generated by counterfeiter"`).

1.5. Forbidden tokens ŌĆö grep each non-skipped file for any of these as whole words / identifiers:
```
\bcontainerName\b
\bContainerChecker\b
\bContainerStopper\b
\bcontainerslot\b
```
Use a single `grep -nE` with alternation: `grep -nE '\bcontainerName\b|\bContainerChecker\b|\bContainerStopper\b|\bcontainerslot\b' "$f"`. Accumulate matches into `OFFENDERS` exactly as the sibling scripts do (via the `<<EOF` heredoc). Do NOT forbid `ContainerCounter` or `containerlock` ŌĆö those are out of scope (genuinely docker concepts).

1.6. Tail: strip the leading newline (`OFFENDERS=$(printf '%s' "$OFFENDERS" | sed '/^$/d')`); if non-empty, print to stderr a message like `hotpath-execution-naming-check: container vocabulary leaked into neutral layer (use executionID / ExecutionChecker / ExecutionStopper / executionslot; container words belong in pkg/executor, pkg/launchpolicy, pkg/containerlock):` followed by the offender list; then `if [ "$MODE" = "strict" ]; then exit 1; fi`; finally `exit 0`.

1.7. Make the script executable (`chmod +x scripts/hotpath-execution-naming-check.sh`) and ensure it passes `shellcheck` (the repo lints shell scripts ŌĆö match the sibling scripts' style to stay clean).

## 3. Leak fixture (demonstrate the failure mode)

3.1. The AC references `scripts/testdata/hotpath-execution-naming/leak.go`. A `.go` file would be processed by `addlicense` and compiled/linted (the `addlicense` target globs `find . -name "*.go"`), which would break the build. Therefore ship the fixture as a NON-compiled file: create `scripts/testdata/hotpath-execution-naming/leak.go.txt` containing a snippet that includes a forbidden token, e.g.:
```
// fixture: demonstrates a container-vocabulary leak the gate must catch.

```
The `.go.txt` extension keeps it out of the Go build and out of `addlicense`'s `*.go` glob.

3.2. Add a self-test of the gate's DETECTION to the script's own behavior: in the script, when invoked as `scripts/hotpath-execution-naming-check.sh selftest`, copy the fixture into a temp file with a `.go` name under a temp dir, run the same forbidden-token grep against it, and assert it matches (exit 1 with a message if the fixture does NOT trigger detection; exit 0 if it does). Keep this mode small and clearly commented. (This proves AC: "the test fixture for the gate demonstrates the failure mode" without polluting the real package tree.)
